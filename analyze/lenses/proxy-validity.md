# Proxy-Validity Lens

Use when estimating an unobservable/latent quantity (ability, credibility,
capability, quality, health) from observable proxies (credentials, speech,
portfolios, demos, PR), or when asked to apply a bias-discount to such an estimate.

1. **Precision honesty first.** State the instrument's actual resolving power
   before the estimate. If the requested precision exceeds it (±5 asked, ±12
   deliverable), deliver both: the requested band labeled as a point-estimate
   band, and the honest CI. Never let the question dictate the precision.
2. **Condition each proxy on the subject.** For every proxy ask: *who was this
   validated on, and what inflates or deflates it for THIS subject?* Standard
   inflators: high SES → credentials; rehearsed domain → apparent fluency;
   group-level trait offsets → any group-loaded proxy; edited/team output →
   individual-skill inference.
3. **Agreement tightens, it doesn't move.** Independent proxies agreeing on a
   value narrow the CI around it; they are not votes to raise it.
4. **Premise-check proposed corrections.** When the user (or a source) proposes
   a bias adjustment: verify the empirical premise, then split direction from
   magnitude — a correction can be directionally right and 10× overstated
   (e.g., "women talk 3× more" is debunked; the real fluency d≈0.2–0.45 is
   worth 1–2 IQ points, not 10). Check the base-rate DIRECTION too: a claimed
   bias may run opposite in the actual data (female founders get ~2% of VC
   dollars — under the bias hypothesis a large female-led raise is a stronger
   signal, not a weaker one).
5. **Use absence as tail evidence.** A missing evidence class (no quantitative
   artifact, no adversarial performance) caps the corresponding tail more than
   additional same-kind positives raise it.
6. **Stopping rule.** The Nth consistent sample of the same evidence KIND adds
   ≈ nothing. Name the evidence class that would actually discriminate; if it
   is unreachable, stop and say so.

Output:

- Estimate: requested band + honest CI, separately labeled
- Proxy table: proxy → validity condition → discount applied
- Corrections ledger: proposed adjustment → premise verified? → direction/magnitude verdict
- Discriminating evidence that would move the estimate (reachable or not)
